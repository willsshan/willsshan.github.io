泛型的PECS原则

假设有这么一个类Stack：

public class Stack<E> {
    private static final int DEFAULT_INITIAL_CAPACITY = 16;
    private E[] elements;
    private int size;

    /**
     * elements数组中只能包含E类型的元素(调用push方法向栈中添加元素)，因此这里的强转是安全的。
     */
    @SuppressWarnings("unchecked")
    public Stack() {
        this.elements = (E[]) new Object[DEFAULT_INITIAL_CAPACITY];
    }

    public void push(E e) {
        ensureCapacity();
        elements[size++] = e;
    }

    public E pop() {
        if (size == 0) {
            throw new EmptyStackException();
        }

        E result = elements[--size];
        // 消除引用，防止内存泄漏
        elements[size] = null;
        return result;
    }

    public boolean isEmpty() {
        return size == 0;
    }

    private void ensureCapacity() {
        if (elements.length == size) {
            elements = Arrays.copyOf(elements, 2 * size + 1);
        }
    }
}

这时我想增加一个putall的方法，如下面所示：

    public void pushAll(Iterable<E> source) {
        for (E e : source) {
            push(e);
        }
    }

进行一个简单的测试：

    public static void main(String[] args) {
        Stack<Number> s1 = new Stack<>();
        Iterable<Number> i1 = new ArrayList<>();
        s1.pushAll(i1);
    }

此时编译是通过的，并没有任何的问题。

如果我想把类型为Integer（Integer是Number的子类）的集合调用putall方法放进去，就会有问题了

    public static void main(String[] args) {
        Stack<Number> s1 = new Stack<>();
        Iterable<Integer> i1 = new ArrayList<>();
        s1.pushAll(i1);//报错
    }

为什么报错，因为Stack生命的类型为E，这里的E声明为Number，而i1为Integer类型，即使它两为继承关系，也不能进行调用。怎么办呢？只要在putall的参数里面进行声明就可以。

    public void pushAll(Iterable<? extends E> source) {
        for (E e : source) {
            push(e);
        }
    }

? extends E说明这里可以用E的子类，也就是说如果E为Number，那么它的子类Integer也是可以正常调用putall方法的。我们再来试一下：

    public static void main(String[] args) {
        Stack<Number> s1 = new Stack<>();
        Iterable<Integer> i1 = new ArrayList<>();
        s1.pushAll(i1);//正常
    }

与? extends E相对应的是？super E，即E的父类，这里可以用E的父类吗？ 如果把putall方法改为如下：

    public void pushAll(Iterable<? super E> source) {
        for (E e : source) {//报错
            push(e);
        }
    }

为什么报错，因为父类是不可以自动向下转型的，特别是不知道父类有几个派生类，有没有解决办法呢？有！

    public void pushAll(Iterable<? super E> source) {
        for (Object o : source) {//正常
            push((E) o);//强制类型转换,有报错的风险
        }
    }

上面是put，下面讲popAll，即取操作。参数为一个消费者“destination”

    public void popAll(Collection<? super E> destination) {
        while (!isEmpty()) {
            destination.add(pop());
        }
    }

同样，进行测试：

    public static void main(String[] args) {
        Stack<Number> s1 = new Stack<>();
        Collection<Object> c1 = new ArrayList<>();
        Collection<Integer> c2 = new ArrayList<>();
        s1.popAll(c2);//出错
        s1.popAll(c1);//正常
    }

经过上面的分析，这里的问题很容易知道为什么，因为参数必须为E的父类。

如果改为extends会怎么样呢：

出错，很明显，pop这个函数只会返回E这个元素，而destination的add方法只会添加E的子类，有人说了，强转不行吗，答案是不行，因为现在根本不知道谁是子类，所以没办法强转。

所以总结一个PECS原则，为什么叫PECS呢？

首先popall的时候，destination的角色是一个消费者，Cousumer即消费了stack中的元素，一般消费者用? super E，而第一个例子中的pushall的参数就是生产者Producer，它的泛型一般是? extends E，即PECS。
