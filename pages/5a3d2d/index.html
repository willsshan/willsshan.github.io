Elasticsearch实现全文检索

# Elasticsearch实现全文检索

# 问题提出：

如何对一个索引库进行全文检索，这里的全文检索意思是用户输入一个值，后台返回与这个值相对应的所有的信息。但是一般使用的querybuild是在一个字段里面进行检索操作，显然不符合我们的需求，这里要对一个库里面的所有的字段进行检索。

# 方法1:

查阅资料，发现multiMatchQuery可以满足我们的需求，其实就是bool的一个变形

# 从kinaba来看multiMatch：

有这样一个索引：

PUT multimatchtest/_mapping/multimatch_test
{
  "properties": {
    "subject": {
      "type": "text"
    },
    "message": {
      "type": "text"
    }
  }
}

GET multimatchtest/multimatch_test/_search
{
  "query": {
    "multi_match": {
      "query": "multimatch",
      "fields": ["subject", "message"]
    }
  }
}

可以看到fileds写了多个字段，这里的multi_match会把待查值分别匹配每个字段，然后把结果汇总到结果集里面。

fields字段可以用匹配符进行描述：

GET multimatchtest/multimatch_test/_search
{
  "query": {
    "multi_match": {
      "query": "multimatch",
      "fields": ["subject", "mess*"]
    }
  }
}

也可以增加每一个权重，方法是加上==^==,后面跟上权重数，一般提高权重就设为大于1的数，降低权重就把权重设为0-1：

GET multimatchtest/multimatch_test/_search
{
  "query": {
    "multi_match": {
      "query": "multimatch",
      "fields": ["subject^3", "mess*"]
    }
  }
}

如果fileds里面什么也不填，默认是检索所有字段，但是注意==在springboot里面的QueryBuilders.multiMatchQuery()如果fileds什么也不填会报错的==。

multi_match还有一个比较重要的参数：type

  • best_fields 是默认类型，会将任何与查询匹配的文档作为结果返回，但是只使用最佳字段的 _score 评分作为评分结果返回。
  • most_fields 将任何与查询匹配的文档作为结果返回，并所有匹配字段的评分合并起来
  • phrase 在 fields 中的每个字段上均执行 match_phrase 查询，并将最佳字段的 _score 作为结果返回
  • phrase_prefix 在 fields 中的字段上均执行 match_phrase_prefix 查询，并将每个字段的分数进行合并

# 从java看multimatch

/**
 * Creates a match query with type "BOOLEAN" for the provided field name and text.
 *
 * @param fieldNames The field names.
 * @param text       The query text (to be analyzed).
 */
public static MultiMatchQueryBuilder multiMatchQuery(Object text, String... fieldNames) {
    return new MultiMatchQueryBuilder(text, fieldNames); // BOOLEAN is the default
}

可以看到这个函数可以直接用，text传入待查参数，fieldNames传入字段名

写完之后使用的使用可能会报错：

"caused_by" : {
            "type" : "number_format_exception",
            "reason" : "For input string: \"\""
}

这是由于这个函数传入的是字符串类型，但是你建立的索引字段不是字符串类型，就会报这个类型错误。这个错误对于这个方法来说，不好解决。。。

# 方法2：

又经过查阅资料，发现了一个更有用的函数queryStringQuery

/**
 * A query that parses a query string and runs it. There are two modes that this operates. The first,
 * when no field is added (using {@link QueryStringQueryBuilder#field(String)}, will run the query once and non prefixed fields
 * will use the {@link QueryStringQueryBuilder#defaultField(String)} set. The second, when one or more fields are added
 * (using {@link QueryStringQueryBuilder#field(String)}), will run the parsed query against the provided fields, and combine
 * them either using Dismax.
 *
 * @param queryString The query string to run
 */
public static QueryStringQueryBuilder queryStringQuery(String queryString) {
    return new QueryStringQueryBuilder(queryString);
}

这个函数简单的来说，就是把字段值，待查值，全部都转为字符串类型，默认就是查所有字段，返回一个结果集。

# 方法3：

使用bool

    BoolQueryBuilder boolQueryBuilder = QueryBuilders.boolQuery()
    .must(QueryBuilders.termQuery("sex", "女"))

    .must(QueryBuilders.rangeQuery("age").gte(30).lte(40))

    .mustNot(QueryBuilders.termQuery("sect.keyword", "明教"))

    .should(QueryBuilders.termQuery("address.keyword", "峨眉山"))

    .should(QueryBuilders.rangeQuery("power.keyword").gte(50).lte(80))

    .minimumShouldMatch(1);  // 设置should至少需要满足几个条件

使用boolquery可以对每个字段进行单独的设置，也可以达到相同的效果。

以上就是在实现初步全文检索时的方法，仅供参考。
